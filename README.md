# Windows 企业IT信创系统信息采集工具

> 当前版本： V1.2.0  
> 运行平台：Windows / .NET Framework 4.8  
> 项目类型：Visual Studio Console Application

`Windows 企业IT信创系统信息采集工具` 面向企业 IT 资产盘点、设备信息复核、信创产品判断和批量采集归档场景设计。工具通过本机 WMI、注册表和本地规则综合检测硬件、系统、网络、外设与信创相关信息，并支持将报告输出为 `txt`、`json`、`csv`，便于一线采集、资产登记和信息化管理员汇总分析。

## 功能亮点

- ◆ **基础摘要**：快速提取品牌、型号、序列号、预计出厂日期、操作系统、CPU、网卡等资产登记核心字段。
- ■ **完整检测**：覆盖操作系统、品牌型号、处理器、内存、硬盘驱动器、显卡、主板 BIOS、网络、音频、电池、显示器、外围设备和信创判定。
- ✓ **复核增强**：增强主板序列号兜底、内存 DDR 代数识别、笔记本网卡状态核实、设备类型判定等复核逻辑。
- ★ **信创判定**：按品牌、型号、CPU、操作系统四类规则评分，输出命中项、判断过程和综合置信度。
- ▣ **批量归集**：支持按部门和门牌号归档，上传 `txt/json/csv` 三类报告，并生成共享目录统计表。
- → **易用体验**：提供彩色终端、动态进度条、实时检测时间、菜单化操作和 WebDAV 连通性预检测。

## 主菜单

```text
1  基础硬件信息检测
2  硬件完整信息检测
3  信创产品判断
4  单模块信息检测
5  系统报告上传
6  退出程序
```

### 1. 基础硬件信息检测

用于快速采集设备登记所需核心字段，包含：

- 计算机名
- 品牌、型号、设备类型
- 序列号、预计出厂日期
- 操作系统、系统版本、激活状态
- CPU
- 网卡信息

基础检测结果默认不显示来源信息，适合一线登记和快速核对。

### 2. 硬件完整信息检测

输出所有模块的完整检测结果，适合排障、资产审计和信息化复核。检测模块包括：

- 操作系统
- 品牌型号
- 处理器
- 内存
- 硬盘驱动器
- 显卡
- 主板和 BIOS
- 网络
- 音频设备
- 电池信息
- 显示器信息
- 外围设备
- 信创判定

### 3. 信创产品判断

信创判断基于本地规则综合评分：

| 维度 | 权重 | 示例依据 |
| --- | ---: | --- |
| 品牌 | 30 | 国产品牌、信创相关品牌关键词 |
| CPU | 35 | 兆芯、海光、飞腾、龙芯、鲲鹏、申威等关键词 |
| 操作系统 | 20 | 统信 UOS、麒麟、openEuler、openKylin 等关键词 |
| 型号 | 15 | 信创、国产化、Kunpeng、Phytium、Hygon 等型号关键词 |

判断结果分为：

- `高度疑似信创设备`
- `疑似信创设备`
- `未达到信创判定条件`

工具会同步输出判断过程、命中项和置信度，便于人工复核。

### 4. 单模块信息检测

支持按模块单独检测，适合 Debug、问题定位或只查看某一类硬件信息。

### 5. 系统报告上传

系统报告上传包含三个入口：

```text
1  完整检测并导出
2  批量采集上传提交        员工使用
3  汇总分析共享目录表      信息化管理员专用
```

## 批量采集上传

批量采集上传用于大批量人员设备信息采集。

### 默认 WebDAV 配置

```text
默认 WebDAV 地址: http://WebDAV服务器内网IP/DeviceReports
默认账号: debug
密码: 运行时输入，不在本地持久化保存
```

工具会在采集前先执行 WebDAV 连通性测试，测试步骤包括：

- OPTIONS 探测服务
- 创建测试目录
- 上传测试文件
- 清理测试文件
- 确认访问测试完成

如果连通性测试失败，工具会提示更换地址或联系信息化管理员，不会继续执行完整硬件扫描，避免局域网外环境下长时间等待后才失败。

### 成功地址记忆

连通性测试成功后，工具会记忆最近一次成功的归集地址，下次进入批量采集时优先引用该地址，并继续保留连通性测试和失败后更换地址逻辑。

本地记忆路径：

```text
%LOCALAPPDATA%\SystemInfoCollector\last_collect_address.txt
```

### 员工采集流程

批量采集上传时，员工需要依次输入：

1. 所在部门  
   可选择预设部门，也可直接输入其他部门名称。
2. 设备使用人员姓名
3. 当前设备使用位置，也就是门牌号

报告归档目录结构：

```text
WebDAV根目录/
└─ 部门/
   └─ 门牌号/
      ├─ 部门-姓名-计算机名-报告日期.txt
      ├─ 部门-姓名-计算机名-报告日期.json
      └─ 部门-姓名-计算机名-报告日期.csv
```

## 报告格式

工具支持三种报告格式：

| 格式 | 用途 |
| --- | --- |
| txt | 人工阅读、现场核查、问题反馈 |
| json | 系统集成、结构化留档 |
| csv | 汇总统计、Excel 分析、资产台账 |

CSV 字段包含简易资产登记所需核心字段，并保留基础检测口径下的网卡 IP 和 MAC 信息。

## 命令行用法

### 批量采集

```powershell
SystemInfoDebug.exe --collect
```

### 指定 WebDAV 地址、账号和密码

```powershell
SystemInfoDebug.exe --collect "http://server:port/DeviceReports" debug password
```

### 汇总采集目录 CSV

```powershell
SystemInfoDebug.exe --merge "\\server\设备信息采集"
```

## 构建方式

### 使用 Visual Studio

1. 打开 `SystemInfoDebug.sln`
2. 选择 `Debug` 或 `Release`
3. 生成解决方案

### 使用 MSBuild

```powershell
msbuild .\SystemInfoDebug.sln `
  /t:Build `
  /p:Configuration=Debug `
  /p:Platform="Any CPU" `
  /v:minimal
```

生成文件示例：

```text
bin\Debug\SystemInfoDebug.exe
```

## 终端体验

工具支持 ANSI/VT 彩色输出，并兼容 Windows Terminal、PowerShell、Git Bash / MINGW64 等常见终端环境。

颜色约定：

- 主标题和标题框：梅红色
- 运行中进程：蓝色
- 正常、完成、命中：绿色，前缀 `✓`
- 需核实、警告：黄色，前缀 `⚠`
- 错误：红色，前缀 `✗`
- 注释说明：灰色

动态进度条采用完整方块填充样式：

```text
模块: 42.0% [████████████░░░░░░░░░░░░]
```

## 注意事项

- 本工具主要依赖 Windows WMI、注册表和本地规则，部分硬件字段是否可读取取决于设备厂商和系统权限。
- WebDAV 密码仅在运行时输入，不做本地持久化保存。
- 信创判断为规则化辅助判断，最终结论建议结合采购台账、设备铭牌和人工复核。
- 批量采集前建议先确认 WebDAV 服务地址、端口、账号权限和网络连通性。

- 信息化管理员共享目录统计汇总
- 现场排障和硬件信息快速导出
